## Tips for Writing Release Notes

1. **Be Clear**: Use simple, clear language
2. **Categorize**: Group features, fixes, and improvements
3. **Be Specific**: Mention what changed and why it matters
4. **Include Installation**: Always include installation steps
5. **Use Emojis**: Makes release notes more engaging (optional)
6. **Version Numbers**: Keep version numbers consistent
7. **Breaking Changes**: Clearly mark any breaking changes
8. **Migration Guide**: If needed, include steps to migrate data
